# 数据炼金术士任务体系全面梳理 - Product Requirement Document

## Overview
- **Summary**: 基于课程内容认知复合理论和刻意练习框架，全面梳理、优化和完善数据炼金术士平台的任务体系，确保题目内容合理、数量充足、判题逻辑严密、难度梯度合理，对学生的学习负责。
- **Purpose**: 解决现有任务体系中存在的题目内容重复、判题逻辑简单、难度梯度不合理、题目数量不足等问题，构建一个科学、严谨、有效的Excel学习任务体系。
- **Target Users**: Excel初学者、中级学习者、希望系统提升Excel技能的学生和职场人士

## Goals
- 梳理和优化所有现有任务，确保每个任务的内容、描述、提示、步骤都清晰准确
- 建立科学的难度分级体系，确保初级、中级、高级任务的难度梯度合理
- 完善判题逻辑，确保能准确验证学生的操作，包括数值、公式、格式等多个维度
- 补充缺失的任务，确保每个技能点都有足够数量和难度的题目
- 基于认知复合理论设计任务，确保知识的逐步递进和复合应用
- 基于刻意练习框架设计反馈机制，确保学生能获得及时、准确的反馈

## Non-Goals (Out of Scope)
- 不添加全新的技能点，仅优化现有12个技能点的任务体系
- 不改变平台的整体UI/UX设计
- 不添加后端服务器功能，继续使用mock数据
- 不实现用户登录和持久化存储功能

## Background & Context
当前数据炼金术士平台有以下问题：
1. 任务内容重复：多个任务使用相似的数据和要求
2. 判题逻辑简单：很多任务只检查是否包含特定函数名，不验证实际计算结果
3. 难度梯度不合理：部分中级任务难度超过高级任务
4. 题目数量不足：部分技能点只有1个任务
5. 反馈不完整：任务验证只返回简单的成功/失败，没有具体的错误提示
6. 缺少复合应用任务：大部分任务只针对单个技能，缺少需要多个技能组合的题目

## Functional Requirements
- **FR-1**: 任务内容优化 - 确保每个任务的描述、提示、步骤都清晰、准确、不重复
- **FR-2**: 难度分级体系 - 建立明确的初级、中级、高级任务判定标准
- **FR-3**: 判题逻辑完善 - 实现能验证数值、公式、格式的多维度判题系统
- **FR-4**: 题目数量补充 - 确保每个技能点至少有3个任务（初级1个、中级1个、高级1个）
- **FR-5**: 复合应用任务 - 设计需要使用多个技能点的综合任务
- **FR-6**: 反馈机制优化 - 提供具体的错误提示和改进建议
- **FR-7**: 任务数据验证 - 确保所有任务的初始数据、目标单元格、验证规则都正确

## Non-Functional Requirements
- **NFR-1**: 学习效果 - 基于认知理论设计，确保学生能有效掌握知识
- **NFR-2**: 可扩展性 - 任务体系设计应便于后续添加新任务
- **NFR-3**: 准确性 - 判题逻辑准确率达到95%以上
- **NFR-4**: 完整性 - 覆盖Excel常用功能的80%以上

## Constraints
- **Technical**: 继续使用现有的React + Univer技术栈
- **Business**: 必须在1周内完成任务体系梳理
- **Dependencies**: 依赖现有的12个技能点分类体系

## Assumptions
- 现有的12个技能点分类是合理的
- 学生从初级任务开始学习，逐步提升到中级、高级
- 学生需要即时、准确的反馈来改进学习
- 复合应用任务能有效提升学生的综合能力

## Acceptance Criteria

### AC-1: 任务内容优化
- **Given**: 平台有24个现有任务
- **When**: 完成任务内容梳理
- **Then**: 所有任务的描述、提示、步骤都清晰准确，没有重复内容
- **Verification**: `human-judgment`
- **Notes**: 需要教育专家审核任务内容的合理性

### AC-2: 难度分级体系
- **Given**: 有初级、中级、高级三个难度等级
- **When**: 完成难度分级标准制定
- **Then**: 每个任务都有明确的难度等级，且难度梯度合理（初级任务的完成时间≤10分钟，中级≤20分钟，高级≤30分钟）
- **Verification**: `programmatic`
- **Notes**: 可以通过任务步骤数和涉及的函数数量来量化难度

### AC-3: 判题逻辑完善
- **Given**: 有任务验证系统
- **When**: 完成判题逻辑优化
- **Then**: 判题系统能验证：1) 目标单元格的数值 2) 使用的公式 3) 单元格格式 4) 数据结构
- **Verification**: `programmatic`
- **Notes**: 需要为每个任务编写详细的验证规则

### AC-4: 题目数量补充
- **Given**: 有12个技能点
- **When**: 完成题目补充
- **Then**: 每个技能点至少有3个任务（初级1个、中级1个、高级1个），总任务数≥36个
- **Verification**: `programmatic`

### AC-5: 复合应用任务
- **Given**: 有完整的技能点体系
- **When**: 完成复合任务设计
- **Then**: 至少有6个复合应用任务，每个任务需要使用≥2个技能点
- **Verification**: `human-judgment`

### AC-6: 反馈机制优化
- **Given**: 有任务提交功能
- **When**: 完成反馈机制优化
- **Then**: 任务验证返回：1) 具体的错误位置 2) 错误原因 3) 改进建议 4) 参考示例
- **Verification**: `human-judgment`

### AC-7: 任务数据验证
- **Given**: 有所有任务数据
- **When**: 完成数据验证
- **Then**: 所有任务的初始数据、目标单元格、验证规则都经过测试验证，确保正确性
- **Verification**: `programmatic`
