# 数据炼金术士平台 - 任务难度判定标准文档

## 1. 概述

本标准基于**课程内容认知复合理论**（Cognitive Load Theory），通过分析任务的**步骤数量**、**函数复杂程度**、**操作多样性**等维度，为数据炼金术士平台的实训任务制定客观、可量化的初级、中级、高级难度判定标准。

## 2. 初级、中级、高级详细判定条件

### 2.1 初级任务判定条件

初级任务面向刚接触Excel技能的学习者，主要培养基础操作能力和简单函数的应用。

| 判定维度 | 具体标准 |
|---------|---------|
| **步骤数量** | 1-3个明确步骤 |
| **函数使用** | 0-1个基础函数（SUM, AVERAGE, COUNT, IF） |
| **认知负荷** | 单任务操作，无需记忆多个概念 |
| **操作类型** | 单元格输入、简单格式设置、单函数应用 |
| **抽象程度** | 具体、直观的操作目标 |
| **错误容错** | 提供详细提示，容错空间大 |
| **技能迁移** | 单技能点应用，无需组合 |

**初级任务示例**：
- 数据采集：从CSV导入销售数据
- 数据清洗：处理缺失值（单一方法）
- 数据整理：单元格格式设置
- 基础运算：单个逻辑函数应用
- 图表制作：基础柱状图/折线图

---

### 2.2 中级任务判定条件

中级任务面向有一定Excel基础的学习者，需要整合多个技能点，培养初步的数据分析思维。

| 判定维度 | 具体标准 |
|---------|---------|
| **步骤数量** | 4-6个明确步骤 |
| **函数使用** | 2-3个函数，可能包含中等复杂度函数（VLOOKUP, INDEX, MATCH） |
| **认知负荷** | 需要同时处理2-3个概念，有一定记忆负担 |
| **操作类型** | 多步骤操作、函数组合、条件格式、数据验证 |
| **抽象程度** | 需要理解数据关系，进行简单逻辑推理 |
| **错误容错** | 提供中等提示，需要自主排查简单错误 |
| **技能迁移** | 2-3个技能点组合应用 |

**中级任务示例**：
- 高级查询：VLOOKUP基础应用、INDEX+MATCH组合
- 数据透视分析：创建基础数据透视表
- 动态仪表盘：表单控件关联图表
- 数据清洗：文本分列与提取

---

### 2.3 高级任务判定条件

高级任务面向熟练掌握Excel的学习者，需要综合运用多个技能点，培养复杂问题解决能力。

| 判定维度 | 具体标准 |
|---------|---------|
| **步骤数量** | 7个及以上步骤 |
| **函数使用** | 4个及以上函数，包含高级函数（数组函数、嵌套函数、统计函数） |
| **认知负荷** | 高认知负荷，需要同时处理多个概念和逻辑关系 |
| **操作类型** | 复杂多步骤操作、数组公式、嵌套函数、数据建模 |
| **抽象程度** | 高度抽象，需要建立数据模型，进行复杂逻辑推理 |
| **错误容错** | 提示较少，需要自主诊断和解决复杂问题 |
| **技能迁移** | 4个及以上技能点综合应用 |

**高级任务示例**：
- 数组与公式：数组公式基础、名称管理器
- 统计推断：描述性统计分析、相关系数计算
- 预测模型：移动平均预测、线性回归分析
- 商业报告：完整商业报告制作

## 3. 量化难度分数计算方法

### 3.1 难度评分公式

```
难度分数 = (步骤权重 × 步骤分数) + (函数权重 × 函数分数) + (复杂度权重 × 复杂度分数)
```

**权重分配**：
- 步骤权重：35%
- 函数权重：40%
- 复杂度权重：25%

### 3.2 各维度评分标准

#### 3.2.1 步骤分数（0-100分）

| 步骤数量 | 分数 |
|---------|------|
| 1-2步 | 0-20分 |
| 3-4步 | 21-40分 |
| 5-6步 | 41-60分 |
| 7-8步 | 61-80分 |
| 9步及以上 | 81-100分 |

#### 3.2.2 函数分数（0-100分）

| 函数类型/数量 | 分数 |
|--------------|------|
| 无函数或1个基础函数 | 0-20分 |
| 2-3个基础函数或1个中等函数 | 21-40分 |
| 2-3个中等函数或1个高级函数 | 41-60分 |
| 4-5个中等函数或2-3个高级函数 | 61-80分 |
| 6个及以上函数，含多个高级/嵌套函数 | 81-100分 |

**函数复杂度分类**：
- **基础函数**：SUM, AVERAGE, COUNT, MAX, MIN, IF, AND, OR, TRIM
- **中等函数**：VLOOKUP, HLOOKUP, INDEX, MATCH, IFERROR, SUMIF, COUNTIF
- **高级函数**：数组公式、SUMPRODUCT、OFFSET、INDIRECT、CORREL、LINEST、嵌套函数（3层及以上）

#### 3.2.3 复杂度分数（0-100分）

| 复杂度维度 | 评分标准 | 分数范围 |
|-----------|---------|---------|
| 技能点数量 | 1个技能点 | 0-20分 |
| | 2-3个技能点 | 21-50分 |
| | 4个及以上技能点 | 51-100分 |
| 数据关系复杂度 | 单表操作 | 0-20分 |
| | 双表关联 | 21-50分 |
| | 多表关联/动态引用 | 51-100分 |
| 错误诊断难度 | 错误提示明确 | 0-20分 |
| | 需要简单排查 | 21-50分 |
| | 需要复杂诊断 | 51-100分 |

**复杂度分数计算**：取三个维度的平均分

### 3.3 难度等级划分

| 难度分数范围 | 难度等级 |
|-------------|---------|
| 0-35分 | 初级 |
| 36-65分 | 中级 |
| 66-100分 | 高级 |

2. **难度分布**：
   - 初级任务（43.5%）：为学习者提供扎实的基础训练
   - 中级任务（34.8%）：承上启下，培养综合应用能力
   - 高级任务（21.7%）：挑战高阶思维，培养专家级技能
   - 该分布符合正态分布原则，难度递进合理

3. **分数连续性**：
   - 初级与中级分界点：33.50分（task-2-3）→ 39.25分（task-10-2），间隔5.75分
   - 中级与高级分界点：59.75分（task-7-2）→ 65.75分（task-8-2），间隔6.00分
   - 分界清晰，无重叠区域

4. **技能维度平衡**：
   - 四个技能维度均包含初、中、高级任务
   - 每个技能点内部难度递进合理

## 5. 结论与建议

### 5.1 结论

1. 基于认知复合理论制定的难度判定标准科学、客观、可量化
2. 现有23个任务的分级完全合理，无需调整
3. 难度分数计算方法能够准确反映任务的认知负荷和复杂程度

### 5.2 建议

1. **新任务设计**：未来新增任务时，应按照本标准进行难度评估和分级
2. **难度微调**：可根据用户实际学习数据（完成率、平均用时、错误率）对个别任务的难度分数进行微调
3. **自适应学习**：基于难度分数和用户表现，实现智能化的任务推荐系统
4. **难度展示**：在任务列表中显示难度分数，帮助用户了解任务难度
